# Бронирование переговорок

Простой сервис бронирования переговорных комнат: регистрация сотрудников по PIN-коду, календарь занятости по дням, админ-панель для управления комнатами.

Технологии: чистые HTML/CSS/JS (без сборки), [Supabase](https://supabase.com) (Postgres) как база данных.

## Структура проекта

```
index.html               — вход/регистрация + календарь бронирования
admin.html                — админ-панель (комнаты, пользователи, брони, модерация доски событий)
events.html                — доска событий (посты + фото офиса)
assets/config.js           — ключи Supabase, пароль админки, настройки Telegram-бота
assets/common.js           — общие утилиты (в т.ч. генерация дат повторения, отправка в Telegram)
assets/app.js               — логика index.html
assets/admin.js             — логика admin.html
assets/events.js            — логика events.html
assets/style.css            — стили (тёплая светлая палитра)
supabase-schema.sql          — базовый SQL: комнаты, пользователи, брони
supabase-migration-2.sql     — миграция: комментарии, повторяющиеся брони, доска событий, storage
```

## Шаг 1. Создать проект в Supabase

1. Зайти на [supabase.com](https://supabase.com), создать бесплатный аккаунт и новый проект.
2. В разделе **SQL Editor** выполнить по очереди **все шесть** файлов:
   - `supabase-schema.sql`
   - `supabase-migration-2.sql` (комментарии, повторяющиеся брони, доска событий, storage)
   - `supabase-migration-3.sql` (цветовые теги постов, комментарии к постам, включение realtime)
   - `supabase-migration-4.sql` (видимость постов по отделам)
   - `supabase-migration-5.sql` (чинит недостающее право на обновление постов)
   - `supabase-migration-6.sql` (справочник отделов — управляется из админки)
3. Перейти в **Project Settings → API**. Скопировать:
   - **Project URL** (без `/rest/v1/` на конце!)
   - **anon public** ключ

## Шаг 2. Вписать ключи в проект

Открыть `assets/config.js` и заменить:

```js
const SUPABASE_URL = 'https://YOUR-PROJECT.supabase.co';
const SUPABASE_ANON_KEY = 'YOUR-ANON-PUBLIC-KEY';
const ADMIN_PASSWORD = 'change-me-123'; // придумай свой пароль для админки
```

⚠️ Это публичный (anon) ключ Supabase — он и так предназначен для использования на фронтенде, но данные в таблицах доступны на чтение/запись любому, у кого есть ссылка на сайт. Для внутреннего корпоративного инструмента это приемлемо; для более серьёзной защиты в будущем можно донастроить RLS-политики и Supabase Auth.

## Шаг 2.1. (Необязательно) Настроить уведомления в Telegram

При создании брони переговорки сайт может слать сообщение в Telegram-чат.

1. В Telegram напиши **@BotFather** → `/newbot` → следуй инструкциям → получишь **токен бота**.
2. Добавь бота в свою группу/чат.
3. Узнай **chat_id**:
   - самый простой способ — добавь в тот же чат бота **@userinfobot** (или похожего), либо
   - напиши боту любое сообщение в чате, затем открой в браузере `https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates` и найди `"chat":{"id": ...}` в ответе
4. Впиши оба значения в `assets/config.js`:

```js
const TELEGRAM_BOT_TOKEN = '123456789:AA...';
const TELEGRAM_CHAT_ID = '-100123456789';
```

Если оставить заглушки как есть — сайт просто не будет отправлять уведомления, всё остальное продолжит работать.

⚠️ Токен бота будет виден в открытом коде сайта. Не используй бота, который подключён к чему-то важному помимо этого чата — в худшем случае кто-то посторонний сможет слать сообщения от его имени в этот чат.

## Как это работает

- **Отделы:** единый список, который создаёт и редактирует админ (раздел «Отделы»). При регистрации сотрудник выбирает отдел из этого списка (не вводит текстом) — это убирает разнобой вроде «Айти» / «IT». В карточке пользователя в админке отдел тоже меняется через выпадающий список; если у пользователя стоит старое «нестандартное» значение (осталось с момен(та до этой миграции), оно подсвечивается отдельным пунктом, чтобы админ мог его поправить.
- **Регистрация:** сотрудник вводит имя, выбирает отдел из списка и придумывает PIN (4–6 цифр). PIN хэшируется (SHA-256) перед отправкой в базу — в открытом виде не хранится.
- **Вход:** выбор своего имени из списка + ввод PIN.
- **Бронирование:** на выбранную дату для каждой комнаты показана линейка времени 10:00–19:00 с занятыми (охра) и свободными (шалфейный зелёный) промежутками. Клик по свободному промежутку открывает форму с шагом в 1 час по умолчанию — время можно скорректировать вручную с точностью до минуты.
- **Повторяющиеся брони:** при создании брони можно выбрать «Каждый день / Каждую неделю / Каждый год» и указать дату, до которой повторять (максимум 366 повторов за раз). Даты, где слот уже занят, автоматически пропускаются, остальные создаются. Отмена повторяющейся брони предлагает выбор — отменить только эту дату или все повторы сразу.
- **Детали и комментарии:** клик по занятому блоку (своему или чужому) открывает карточку с деталями брони и комментариями — комментировать может любой зарегистрированный пользователь. Отменить бронь может только тот, кто её создал.
- **Telegram-уведомления:** при создании (или серии) брони в чат уходит сообщение с комнатой, автором, датой(ами) и целью — если заполнены `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID` в `config.js`.
- **Доска событий (`events.html`):** свободное пространство, куда любой зарегистрированный сотрудник может опубликовать текст и/или фото, а также поставить цветовой тег (Новость / Важное / Праздник / Движ). По умолчанию пост виден всем; ограничить видимость конкретными отделами может только админ — через кнопку «Видимость» в таблице модерации на `admin.html` (обычные пользователи такой настройки не видят). Клик по карточке открывает пост целиком с комментариями. Автор может удалить свой пост; админ может удалить любой пост через админ-панель.
- **Реальное время:** календарь броней и доска событий обновляются автоматически, без перезагрузки страницы, когда кто-то другой создаёт бронь, комментирует или публикует пост — через Supabase Realtime.
- **Админ-панель** (`admin.html`, защищена паролем из `config.js`): добавление/редактирование/скрытие/удаление комнат, редактирование имени и отдела любого пользователя, просмотр и отмена любых броней по дате, модерация доски событий.

## Возможные доработки

- Ограничить рабочие часы и шаг сетки настройкой из админки (сейчас — константы `WORK_START` / `WORK_END` в `assets/common.js`).
- Полноценная защита админки через Supabase Auth вместо статического пароля.
- Более гибкие правила повторения (например, «по будням» или «каждые N дней»).
